When you enable AI summaries, the extractor sends each task's fields to your chosen AI backend and appends a concise 1â€“2 sentence first-person status blurb into the `Notes` column (e.g. "I completed the initial review and need to address two open comments."). Tasks without a due date also receive an AI-estimated **ETA** from the same source. Both passes run with bounded concurrency and gracefully fall back to non-AI content when the provider is unavailable.

## Choose an AI source

Use `--ai-source` to pick the backend that generates summaries (and AI-powered ETAs).

<ParamField path="--ai-source" type="string" default="Claude">
  Accepted values: `Claude`, `Gemini`, `ClickUp`, `Both`.
</ParamField>

| Value | What it uses for summaries | What it uses for ETA | API key required? |
|---|---|---|---|
| `Claude` (default) | Local `claude` CLI (Claude Code) in headless print mode, billed against your Max/Pro **OAuth subscription** | Claude (`get_claude_eta`) | No â€” uses your Claude Code login |
| `Gemini` | Google Gemini API with tiered model fallback | Gemini | Yes â€” Google Gemini API key |
| `ClickUp` | The task's existing ClickUp `Summary` custom field | **Deterministic** priority/status estimate | No |
| `Both` | ClickUp `Summary` field first, then falls back to **Claude** | Claude | No |

```bash
# Default â€” Claude via your Max subscription, no API key needed
python main.py --ai-summary

# Explicit Claude
python main.py --ai-summary --ai-source Claude

# Gemini (requires a Gemini API key)
python main.py --ai-summary --ai-source Gemini --gemini-api-key YOUR_KEY

# ClickUp Summary field only â€” no AI calls
python main.py --ai-summary --ai-source ClickUp

# ClickUp field when available, Claude as fallback
python main.py --ai-summary --ai-source Both
```

<Note>
  If you run without `--ai-summary`, the extractor prompts you at startup. Claude is the default â€” pressing Enter accepts it and no key is required. The extractor only loads or prompts for a Gemini key when you pick `Gemini`.
</Note>

## Claude (Max OAuth) â€” default source

The `Claude` source shells out to the local [`claude` CLI](https://docs.claude.com/en/docs/claude-code) in headless print mode (`claude -p`) and supplies each task's fields over stdin. The CLI authenticates with your Claude Code OAuth, so each call bills your **Max or Pro subscription**. There is no Anthropic API key to manage and no Gemini-style free-tier rate limit.

To use it, install Claude Code and sign in once with `claude login`. After that, the extractor finds the CLI on `PATH` and calls it automatically.

<Tip>
  Even if `ANTHROPIC_API_KEY` or `ANTHROPIC_AUTH_TOKEN` are set in your shell, the extractor scrubs them from the `claude` subprocess environment so summaries always bill the subscription, not the API.
</Tip>

### Claude environment variables

| Variable | Default | Purpose |
|---|---|---|
| `CLAUDE_SUMMARY_MODEL` | `claude-haiku-4-5-20251001` | Model id passed to `claude --model`. Accepts a full id or a CLI alias (`haiku`, `sonnet`, `opus`). Reused for ETA estimation. |
| `CLAUDE_SUMMARY_TIMEOUT` | `120` | Per-call timeout in seconds. The extractor falls back to non-AI content if the CLI hangs longer. |

```bash
# Use a heavier model for higher-quality summaries
export CLAUDE_SUMMARY_MODEL=sonnet
python main.py --ai-summary
```

### Claude fallback behaviour

The Claude path always degrades gracefully â€” your export completes even if Claude is entirely unavailable:

| Situation | Behaviour |
|---|---|
| `claude` CLI not found on `PATH` | Prints a one-time warning, falls back to the raw task content for the rest of the run. |
| CLI exits with a non-zero status or an unexpected error | Logs the error and uses the task's original content for that task. |
| Call exceeds `CLAUDE_SUMMARY_TIMEOUT` | Logs a timeout and falls back to the raw content for that task. |
| Subscription **usage limit** hit (rate-limit / "usage limit" message in stderr) | Disables further Claude calls for the rest of the run and short-circuits any queued summaries; remaining tasks use the raw content. |

## Gemini

`--ai-source Gemini` keeps the original Google Gemini path. It still requires a Gemini API key â€” see [Authentication](/clickup/authentication#gemini-api-key-for-ai-summaries) for every supported way to supply one.

The default Gemini model is `gemini-2.5-flash-lite`. Override it by setting `GEMINI_MODEL`:

```bash
export GEMINI_MODEL=gemini-2.5-flash
python main.py --ai-summary --ai-source Gemini --gemini-api-key YOUR_KEY
```

The extractor detects rate limits via HTTP `429` responses, `RESOURCE_EXHAUSTED` errors, and keywords such as `quota`, `rate limit`, `rpm`, and `overload` in the error message. On a rate-limit error it backs off for `initial_delay Ă— 2^attempt` seconds, up to two retries. If every retry fails, it disables further Gemini calls for the rest of the run.

## Concurrent generation

Summaries are generated in a single bounded-concurrency pass **after** all tasks are gathered, not inline during task processing. This typically cuts wall-clock time on large exports by ~3Ă—. Output order is preserved, and once a provider hits a usage or rate limit, queued calls short-circuit instead of waiting.

<ParamField path="AI_SUMMARY_CONCURRENCY" type="integer" default="4">
  Maximum number of summaries (and ETAs) generated in parallel. Clamped to the task count. Applies to both Claude and Gemini.
</ParamField>

```bash
# Speed up a large export
export AI_SUMMARY_CONCURRENCY=8
python main.py --ai-summary

# Be gentler on rate limits / slow machines
export AI_SUMMARY_CONCURRENCY=2
python main.py --ai-summary
```

## AI-powered ETA estimation

Tasks **without a due date** receive an estimated ETA in a second bounded-concurrency pass that runs after summaries. The pass reuses the same `--ai-source` dispatch and `AI_SUMMARY_CONCURRENCY` worker count.

| `--ai-source` | ETA source |
|---|---|
| `Claude` (default) | Claude CLI ETA |
| `Both` | Claude CLI ETA |
| `Gemini` | Gemini ETA |
| `ClickUp` | Deterministic priority/status fallback |

The extractor first assigns every task a deterministic baseline ETA from its priority and status, so the AI pass only **upgrades** due-date-less tasks. If AI is unavailable for any reason (CLI missing, errors, timeout, usage limit), the deterministic baseline remains in place and `calculate_eta()` always returns a valid value.

<Note>
  The `ClickUp` source uses the deterministic ETA because ClickUp itself does not expose an ETA custom field to read.
</Note>

## Example commands

```bash
# Minimal: default Claude source, no API key needed
python main.py --ai-summary

# Default source, HTML output, this week only
python main.py --ai-summary --output-format HTML --date-filter ThisWeek

# Force Gemini, Markdown output, all open tasks
python main.py --ai-summary --ai-source Gemini --gemini-api-key YOUR_KEY --output-format Markdown

# ClickUp AI field only â€” no AI calls and deterministic ETA
python main.py --ai-summary --ai-source ClickUp

# Crank up parallelism on a large export
AI_SUMMARY_CONCURRENCY=8 python main.py --ai-summary
```

<Tip>
  Run with `--ai-source ClickUp` during development to avoid consuming Claude subscription usage or Gemini quota while iterating on other settings.
</Tip>
