The extractor resolves your ClickUp API key automatically by working through a priority chain â€” it checks each source in turn and stops as soon as it finds a valid key. If every automated source fails, it falls back to a secure interactive prompt so you are never locked out.

<Tip>
  The fastest way to configure every variable on this page is to copy `.env.example` to `.env` and fill in your own values. The extractor **auto-loads `.env` (next to `main.py`) on every CLI run**, so you no longer need to export the variables in each shell. Real environment variables and CLI flags still take precedence, so you can override any value at runtime. The weekly Google Sheets sync (`kfj_task_extractor.py`) does the same for `.env.kfj` â€” see [Project-local `.env` auto-load](#project-local-env-auto-load) below. Both files are gitignored once copied.
</Tip>

## Get your ClickUp API key

<Steps>
  <Step title="Open your ClickUp settings">
    Click your avatar in the lowerâ€‘left corner of the ClickUp web app, then select **Settings**.
  </Step>
  <Step title="Navigate to the Apps section">
    In the left sidebar choose **Apps**. Scroll to the **API Token** section.
  </Step>
  <Step title="Copy your token">
    Click **Generate** if no token exists, then copy the displayed value. Treat this token like a password â€” it grants full API access to your account.
  </Step>
</Steps>

## Provide your key

The extractor checks these sources in the order shown below.

<Tabs>
  <Tab title="CLI flag">
    Pass your key directly when you invoke the script. This is the highestâ€‘priority source and overrides everything else:

    ```bash
    python main.py --api-key YOUR_CLICKUP_API_KEY
    ```

    <Warning>
      Do not hardcode your API key in scripts that are committed to version control. Use an environment variable or 1Password for persistent storage.
    </Warning>
  </Tab>

  <Tab title="Environment variable">
    Export `CLICKUP_API_KEY` before running the script. The extractor reads it automatically on every invocation:

    ```bash
    # macOS / Linux
    export CLICKUP_API_KEY=YOUR_CLICKUP_API_KEY
    python main.py

    # Windows PowerShell
    $env:CLICKUP_API_KEY = "YOUR_CLICKUP_API_KEY"
    python main.py

    # Windows Command Prompt
    set CLICKUP_API_KEY=YOUR_CLICKUP_API_KEY
    python main.py
    ```

    <Tip>
      Add the `export` line to your shell profile (e.g. `~/.zshrc` or `~/.bashrc`) so the variable is always present without manual setup.
    </Tip>
  </Tab>

  <Tab title="1Password">
    The extractor integrates with 1Password at three levels, tried in this order:

    **1Password Environment (recommended for teams)**

    Store `CLICKUP_API_KEY` as a variable inside a 1Password Environment, then export its ID:

    ```bash
    export OP_ENVIRONMENT_ID=YOUR_ENVIRONMENT_ID
    # Optional: target a specific account
    export OP_ACCOUNT_NAME=my.1password.com
    python main.py
    ```

    The extractor uses the 1Password Python SDK with Desktop App auth first, then falls back to `op environment read` via the CLI.

    **1Password SDK (service accounts / automation)**

    Provide a service account token to resolve vaultâ€‘based secret references without the desktop app:

    ```bash
    export OP_SERVICE_ACCOUNT_TOKEN=YOUR_SERVICE_ACCOUNT_TOKEN
    python main.py
    ```

    **1Password CLI**

    If neither of the above is configured, the extractor calls `op read` directly. Make sure the `op` binary is in your `PATH` and your account is signed in. Point the extractor at your vault item by setting `CLICKUP_API_SECRET_REFERENCE` (and, for AI summaries, `GEMINI_API_SECRET_REFERENCE`) to the `op://` URI of the credential:

    ```bash
    export CLICKUP_API_SECRET_REFERENCE="op://<vault>/<item>/credential"
    export GEMINI_API_SECRET_REFERENCE="op://<vault>/<item>/credential"
    python main.py
    ```

    When either reference is empty, the extractor skips the 1Password lookup for that key and falls back to `CLICKUP_API_KEY` / `--api-key` (or the equivalent Gemini source).

    <Note>
      If you set `OP_ENVIRONMENT_ID`, the extractor reads from that Environment and does **not** fall through to vaultâ€‘based secret references. Remove `OP_ENVIRONMENT_ID` if you want the vault path instead.
    </Note>
  </Tab>
</Tabs>

## Full authentication priority order

| Priority | Source | Requirement |
|---|---|---|
| 1 | `--api-key` CLI flag | Key passed at runtime |
| 2 | `CLICKUP_API_KEY` environment variable | Variable exported in shell |
| 3 | 1Password Environment (SDK then CLI) | `OP_ENVIRONMENT_ID` set |
| 4 | 1Password SDK (service account) | `OP_SERVICE_ACCOUNT_TOKEN` set |
| 5 | 1Password CLI (`op read`) | `op` installed and signed in, `CLICKUP_API_SECRET_REFERENCE` set |
| 6 | Interactive prompt | Fallback â€” key entered at runtime |

## Project-local `.env` auto-load

Instead of exporting `CLICKUP_WORKSPACE_NAME`, `CLICKUP_SPACE_NAME`, `OP_ENVIRONMENT_ID`, and other settings in every shell, you can drop them into a `.env` file next to the script and the extractor will read them at startup.

**When to use it**

- You run the extractor from multiple shells, an IDE, or a scheduled job and want the same configuration applied everywhere.
- You want workspace/space defaults, 1Password Environment IDs, and secret-reference pointers persisted with the project rather than your shell profile.
- You want to keep CLI flags as a one-off override without rewriting your environment.

**How it works**

On every CLI invocation, `main.py` reads `.env` from the directory containing the script and `kfj_task_extractor.py` reads `.env.kfj` from the same location. The loader runs before any configuration is evaluated, so values like `CLICKUP_WORKSPACE_NAME` and `OP_ENVIRONMENT_ID` apply on a plain `python main.py` even when the shell has not exported them. The loader:

- Skips blank lines, comments (`#`), and malformed entries.
- Accepts an optional `export ` prefix on each line.
- Strips matching single or double quotes around values.
- **Never overrides** variables that are already set in the real environment or values passed via CLI flags.
- Does nothing when the file is missing â€” it's a no-op, not an error.

The loader has no external dependencies, so it works without installing `python-dotenv`.

```bash title=".env (next to main.py)"
# Workspace + space defaults
CLICKUP_WORKSPACE_NAME="Acme"
CLICKUP_SPACE_NAME='Engineering'

# 1Password Environment for CLICKUP_API_KEY / GEMINI_API_KEY
export OP_ENVIRONMENT_ID=YOUR_ENVIRONMENT_ID

# Or a literal API key (kept out of git because .env is gitignored)
# CLICKUP_API_KEY=pk_...
```

```bash
# Picks up .env automatically â€” no shell exports needed.
python main.py

# CLI flag still wins over .env.
python main.py --workspace "Other Workspace"
```

### Secret-safe `.env.kfj` for the weekly Google Sheets sync

`kfj_task_extractor.py` auto-loads `.env.kfj` with the same rules, with one extra safeguard for 1Password references.

The loader **skips `op://` values for `CLICKUP_API_KEY` and `GOOGLE_SHEETS_CREDENTIALS_JSON`**, because those variables are expected to hold the resolved secret at runtime (not the literal `op://` URI). Leaving them in `.env.kfj` is still useful: `op run --env-file=.env.kfj` and the script's built-in 1Password chain resolve them into real credentials before the script uses them.

Other variables load normally, including:

- Plain configuration like `KFJ_LIST_ID`, `KFJ_SHEET_ID`, `KFJ_TAB_NAME`.
- `KFJ_*_SECRET_REFERENCE` pointer variables (for example `KFJ_GOOGLE_SHEETS_CREDENTIALS_SECRET_REFERENCE`), whose values are themselves `op://` URIs that the 1Password chain consumes.
- Literal (non-`op://`) values for `CLICKUP_API_KEY` or `GOOGLE_SHEETS_CREDENTIALS_JSON` if you choose to store them directly.

```bash title=".env.kfj (next to kfj_task_extractor.py)"
KFJ_LIST_ID=901234567890
KFJ_SHEET_ID=1AbCdEf...
KFJ_TAB_NAME="Weekly KFJ"

# Skipped by the auto-loader â€” resolved by op run / the 1Password chain.
CLICKUP_API_KEY=op://Private/ClickUp/credential
GOOGLE_SHEETS_CREDENTIALS_JSON=op://Private/GSheets/credential

# Loaded normally â€” pointer variable consumed by the 1Password chain.
KFJ_GOOGLE_SHEETS_CREDENTIALS_SECRET_REFERENCE="op://Private/GSheets/credential"
```

```bash
# Plain run auto-loads .env.kfj; the 1Password chain resolves the op:// secrets.
python kfj_task_extractor.py

# Or inject everything explicitly via 1Password.
op run --env-file=.env.kfj -- python kfj_task_extractor.py
```

## Workspace selection

The extractor reads workspace and space selection from environment variables or CLI flags, so the same code works against any ClickUp account.

| Variable | Equivalent flag | Purpose |
|---|---|---|
| `CLICKUP_WORKSPACE_NAME` | `--workspace` | Workspace (team) name to extract from. |
| `CLICKUP_SPACE_NAME` | `--space` | Space within the workspace to pull tasks from. |
| `CLICKUP_TEAM_ID` | â€” | Numeric team/workspace ID. Used as a fallback when the workspace name cannot be resolved via the API; otherwise the extractor prompts. |
| `CLICKUP_AI_SUMMARY_FIELD_ID` | â€” | Optional override for the ClickUp custom-field ID used as the "Summary" field. Leave unset to use the built-in default. |

CLI flags take precedence over environment variables. Any value left unset falls back to an interactive prompt at runtime.

```bash
export CLICKUP_WORKSPACE_NAME="Acme"
export CLICKUP_SPACE_NAME="Engineering"
python main.py
```

## Gemini API key (for AI summaries)

When you enable AI summaries with `--ai-summary`, the extractor needs a Google Gemini API key. It uses the exact same priority chain:

<Tabs>
  <Tab title="CLI flag">
    ```bash
    python main.py --ai-summary --gemini-api-key YOUR_GEMINI_API_KEY
    ```
  </Tab>

  <Tab title="Environment variable">
    ```bash
    export GEMINI_API_KEY=YOUR_GEMINI_API_KEY
    python main.py --ai-summary
    ```

    <Note>
      `GEMINI_API_KEY` is the variable name expected inside a 1Password Environment. The extractor maps the secret name "Gemini API key" to this variable automatically.
    </Note>
  </Tab>

  <Tab title="1Password">
    Store `GEMINI_API_KEY` in the same 1Password Environment as `CLICKUP_API_KEY`. Both keys are loaded in a single authentication step when `OP_ENVIRONMENT_ID` is set:

    ```bash
    export OP_ENVIRONMENT_ID=YOUR_ENVIRONMENT_ID
    python main.py --ai-summary
    ```

    If the Environment method is not configured, the extractor falls back to the vault secret reference defined in `GEMINI_API_SECRET_REFERENCE` (an `op://<vault>/<item>/credential` URI). Leave that variable empty to skip the 1Password lookup entirely and rely on `--gemini-api-key` / `GEMINI_API_KEY`.

    ```bash
    export GEMINI_API_SECRET_REFERENCE="op://<vault>/<item>/credential"
    python main.py --ai-summary
    ```
  </Tab>
</Tabs>

<Warning>
  Never hardcode API keys directly in source files or commit them to version control. Use environment variables or a secrets manager such as 1Password.
</Warning>
