The extractor resolves your ClickUp API key automatically by working through a priority chain — it checks each source in turn and stops as soon as it finds a valid key. If every automated source fails, it falls back to a secure interactive prompt so you are never locked out.

<Tip>
  The fastest way to configure every variable on this page is to copy `.env.example` to `.env` and fill in your own values. The repo ships `.env.example` (for `main.py`) and `.env.kfj.example` (for the weekly Google Sheets sync) as the canonical reference for every supported variable. Both files are gitignored once copied.
</Tip>

## Get your ClickUp API key

<Steps>
  <Step title="Open your ClickUp settings">
    Click your avatar in the lower‑left corner of the ClickUp web app, then select **Settings**.
  </Step>
  <Step title="Navigate to the Apps section">
    In the left sidebar choose **Apps**. Scroll to the **API Token** section.
  </Step>
  <Step title="Copy your token">
    Click **Generate** if no token exists, then copy the displayed value. Treat this token like a password — it grants full API access to your account.
  </Step>
</Steps>

## Provide your key

The extractor checks these sources in the order shown below.

<Tabs>
  <Tab title="CLI flag">
    Pass your key directly when you invoke the script. This is the highest‑priority source and overrides everything else:

    ```bash
    python main.py --api-key YOUR_CLICKUP_API_KEY
    ```

    <Warning>
      Do not hardcode your API key in scripts that are committed to version control. Use an environment variable or 1Password for persistent storage.
    </Warning>
  </Tab>

  <Tab title="Environment variable">
    Export `CLICKUP_API_KEY` before running the script. The extractor reads it automatically on every invocation:

    ```bash
    # macOS / Linux
    export CLICKUP_API_KEY=YOUR_CLICKUP_API_KEY
    python main.py

    # Windows PowerShell
    $env:CLICKUP_API_KEY = "YOUR_CLICKUP_API_KEY"
    python main.py

    # Windows Command Prompt
    set CLICKUP_API_KEY=YOUR_CLICKUP_API_KEY
    python main.py
    ```

    <Tip>
      Add the `export` line to your shell profile (e.g. `~/.zshrc` or `~/.bashrc`) so the variable is always present without manual setup.
    </Tip>
  </Tab>

  <Tab title="1Password">
    The extractor integrates with 1Password at three levels, tried in this order:

    **1Password Environment (recommended for teams)**

    Store `CLICKUP_API_KEY` as a variable inside a 1Password Environment, then export its ID:

    ```bash
    export OP_ENVIRONMENT_ID=YOUR_ENVIRONMENT_ID
    # Optional: target a specific account
    export OP_ACCOUNT_NAME=my.1password.com
    python main.py
    ```

    Setting `OP_ENVIRONMENT_ID` alone opts you into the Environment lookup — you do **not** also need to set `CLICKUP_API_SECRET_REFERENCE` or `GEMINI_API_SECRET_REFERENCE`. An empty secret reference now means "look the key up in the Environment by its variable name" (`CLICKUP_API_KEY`, `GEMINI_API_KEY`).

    The extractor authenticates via the 1Password Python SDK using Desktop App auth, or a service account token if one is configured. If you are running the beta `op` CLI with native Environments support, it can also re-exec under `op run --environments <id>`. On a stable `op` (e.g. 2.34.1) the extractor detects that the flag is unsupported and falls through to the SDK path instead, so the beta CLI is not required.

    This path also works on Windows: the previous subprocess hang and the spurious virtualenv re-exec have both been fixed.

    **1Password SDK (service accounts / automation)**

    Provide a service account token to resolve vault‑based secret references without the desktop app:

    ```bash
    export OP_SERVICE_ACCOUNT_TOKEN=YOUR_SERVICE_ACCOUNT_TOKEN
    python main.py
    ```

    **1Password CLI**

    If neither of the above is configured, the extractor calls `op read` directly. Make sure the `op` binary is in your `PATH` and your account is signed in. Point the extractor at your vault item by setting `CLICKUP_API_SECRET_REFERENCE` (and, for AI summaries, `GEMINI_API_SECRET_REFERENCE`) to the `op://` URI of the credential:

    ```bash
    export CLICKUP_API_SECRET_REFERENCE="op://<vault>/<item>/credential"
    export GEMINI_API_SECRET_REFERENCE="op://<vault>/<item>/credential"
    python main.py
    ```

    When either reference is empty *and* `OP_ENVIRONMENT_ID` is not set, the extractor skips the 1Password lookup for that key and falls back to `CLICKUP_API_KEY` / `--api-key` (or the equivalent Gemini source). If `OP_ENVIRONMENT_ID` is set, an empty reference instead means "resolve the key from the Environment by its variable name".

    <Note>
      If you set `OP_ENVIRONMENT_ID`, the extractor reads from that Environment and does **not** fall through to vault‑based secret references. Remove `OP_ENVIRONMENT_ID` if you want the vault path instead.
    </Note>
  </Tab>
</Tabs>

## Full authentication priority order

| Priority | Source | Requirement |
|---|---|---|
| 1 | `--api-key` CLI flag | Key passed at runtime |
| 2 | `CLICKUP_API_KEY` environment variable | Variable exported in shell |
| 3 | 1Password Environment (SDK, with optional beta-CLI re-exec) | `OP_ENVIRONMENT_ID` set (no secret reference required) |
| 4 | 1Password SDK (service account) | `OP_SERVICE_ACCOUNT_TOKEN` set |
| 5 | 1Password CLI (`op read`) | `op` installed and signed in, `CLICKUP_API_SECRET_REFERENCE` set |
| 6 | Interactive prompt | Fallback — key entered at runtime |

## Workspace selection

The extractor reads workspace and space selection from environment variables or CLI flags, so the same code works against any ClickUp account.

| Variable | Equivalent flag | Purpose |
|---|---|---|
| `CLICKUP_WORKSPACE_NAME` | `--workspace` | Workspace (team) name to extract from. When unset or unmatched, the extractor lists your workspaces and prompts you to pick one (auto-selecting if you only belong to one). |
| `CLICKUP_SPACE_NAME` | `--space` | Space within the workspace to pull tasks from. When unset or unmatched, the extractor lists the available spaces and prompts you to pick one. |
| `CLICKUP_TEAM_ID` | — | Numeric team/workspace ID. Used only as a last-resort fallback when the `/team` endpoint genuinely fails to return any workspaces. |
| `CLICKUP_AI_SUMMARY_FIELD_ID` | — | Optional override for the ClickUp custom-field ID used as the "Summary" field. Leave unset to use the built-in default. |

CLI flags take precedence over environment variables. Both `--workspace` and `--space` are optional — leave either unset and the extractor offers an interactive picker at runtime.

```bash
export CLICKUP_WORKSPACE_NAME="Acme"
export CLICKUP_SPACE_NAME="Engineering"
python main.py
```

## Gemini API key (for AI summaries)

When you enable AI summaries with `--ai-summary`, the extractor needs a Google Gemini API key. It uses the exact same priority chain:

<Tabs>
  <Tab title="CLI flag">
    ```bash
    python main.py --ai-summary --gemini-api-key YOUR_GEMINI_API_KEY
    ```
  </Tab>

  <Tab title="Environment variable">
    ```bash
    export GEMINI_API_KEY=YOUR_GEMINI_API_KEY
    python main.py --ai-summary
    ```

    <Note>
      `GEMINI_API_KEY` is the variable name expected inside a 1Password Environment. The extractor maps the secret name "Gemini API key" to this variable automatically.
    </Note>
  </Tab>

  <Tab title="1Password">
    Store `GEMINI_API_KEY` in the same 1Password Environment as `CLICKUP_API_KEY`. Both keys are loaded in a single authentication step when `OP_ENVIRONMENT_ID` is set:

    ```bash
    export OP_ENVIRONMENT_ID=YOUR_ENVIRONMENT_ID
    python main.py --ai-summary
    ```

    With `OP_ENVIRONMENT_ID` set, you do not need to also set `GEMINI_API_SECRET_REFERENCE` — the extractor resolves `GEMINI_API_KEY` from the Environment by name. If the Environment method is not configured, it falls back to the vault secret reference defined in `GEMINI_API_SECRET_REFERENCE` (an `op://<vault>/<item>/credential` URI). When neither variable is set, the extractor skips the 1Password lookup entirely and uses `--gemini-api-key` or `GEMINI_API_KEY` instead.

    ```bash
    export GEMINI_API_SECRET_REFERENCE="op://<vault>/<item>/credential"
    python main.py --ai-summary
    ```
  </Tab>
</Tabs>

<Warning>
  Never hardcode API keys directly in source files or commit them to version control. Use environment variables or a secrets manager such as 1Password.
</Warning>
