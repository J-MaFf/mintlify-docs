The printer scripts handle the three most common printer support scenarios in a Windows environment: silently installing a defined set of network printers for a user, diagnosing the notorious error 0x0000006ba, and restarting a hung Print Spooler service. All three scripts require administrator rights.

<Warning>
  All scripts in this section require an **elevated (Administrator) PowerShell session**. Running them without elevation will either produce an error or produce incomplete results.
</Warning>

---

## addNetworkPrinters.ps1

Silently adds a list of network printers to the current user's profile. The script uses a **marker file** in the user's profile folder (`%USERPROFILE%\printers_added.txt`) to ensure printers are only installed once — ideal for login scripts or GPO deployments.

### Setup

Create a `printerNames.txt` file in the same directory as the script. Add one UNC printer path per line:

```text
\\printserver01\HP-LaserJet-Finance
\\printserver01\Xerox-Color-Reception
\\printserver02\Canon-MFP-IT
```

### Usage

```powershell
# Run from the Printers directory — reads printerNames.txt automatically
.\Scripts\Printers\addNetworkPrinters.ps1
```

The script:
1. Checks for `printerNames.txt` beside the script.
2. Checks for the marker file `%USERPROFILE%\printers_added.txt`.
3. If the marker file **does not exist**, installs every printer in the list using `Add-Printer -ConnectionName` and then creates the marker file.
4. If the marker file **already exists**, skips installation and exits cleanly.
5. Writes a full transcript to `addNetworkPrinters_Transcript.txt` beside the script.

```powershell
# Force a re-install by removing the marker file first
Remove-Item -Path "$env:USERPROFILE\printers_added.txt" -Force
.\Scripts\Printers\addNetworkPrinters.ps1
```

<Note>
  Each user has their own marker file, so running this script as different users on the same machine will install printers correctly for each account. The transcript log is shared per-directory, so runs from multiple accounts will overwrite the same transcript file.
</Note>

## Diagnose-PrintError.ps1

A comprehensive diagnostic script for the Windows print error **0x0000006ba** ("The print spooler service is not running"). It checks local and remote services, firewall rules, network connectivity, and recent event log entries, then prints a categorized remediation guide.

### Usage

```powershell
# Run without a print server argument to check local state only
.\Scripts\Printers\Diagnose-PrintError.ps1

# Include a print server to test connectivity and remote spooler status
.\Scripts\Printers\Diagnose-PrintError.ps1 -PrintServer 'PRINTSERVER01'

# Use an IP address if DNS resolution is the suspected problem
.\Scripts\Printers\Diagnose-PrintError.ps1 -PrintServer '192.168.1.100'
```

### What the Script Checks

<Steps>
  <Step title="Local services">
    Verifies that the **Print Spooler** (`Spooler`) service and the RPC services (`RpcSs`, `RpcEptMapper`) are running on the local machine.
  </Step>
  <Step title="Print server connectivity">
    When `-PrintServer` is supplied, pings the server and tests TCP ports 135, 445, and 139.
  </Step>
  <Step title="Remote Spooler service">
    Checks whether the Spooler service is running on the remote print server.
  </Step>
  <Step title="Local printer connections">
    Lists all network printer connections and their reported status.
  </Step>
  <Step title="Windows Firewall">
    Checks whether the **File and Printer Sharing** firewall rules are enabled on the local machine.
  </Step>
  <Step title="Event log">
    Searches the System event log for Spooler-related events (IDs 7031, 7032, 7034, 7036) from the past 24 hours.
  </Step>
  <Step title="Remediation guide">
    Prints five categorized fix suggestions: restart services, clear the print queue, check the print server, reset printer connections, and network troubleshooting steps.
  </Step>
</Steps>

Each check is timestamped and color-coded — green `[✓]` for passing, red `[✗]` for failures, yellow `[!]` for warnings, and cyan `[i]` for informational output.

## Repair-PrintSpooler.ps1

A focused script that checks the Print Spooler service status and restarts it if it is stopped or in a failed state. Use this as a quick fix when the spooler has hung without needing to run the full diagnostic.

### Usage

```powershell
# Check and repair the spooler silently
.\Scripts\Printers\Repair-PrintSpooler.ps1

# Show detailed step-by-step output
.\Scripts\Printers\Repair-PrintSpooler.ps1 -Verbose
```

### What the Script Does

```powershell
# Equivalent manual steps the script automates:

# 1. Check current status
Get-Service -Name 'Spooler'

# 2. If not running, stop any intermediate state and restart
Stop-Service -Name 'Spooler' -Force
Start-Service -Name 'Spooler'

# 3. Confirm it is running
Get-Service -Name 'Spooler'
```

The script waits up to 30 seconds for the service to reach the `Stopped` state before restarting, and another 30 seconds for it to reach `Running`. If either timeout is exceeded, it exits with error code `1` and a clear message.

**Exit codes:**

| Code | Meaning |
|---|---|
| `0` | Spooler is running (already was or successfully restarted) |
| `1` | Failed to repair — manual intervention required |

<Tip>
  If `Repair-PrintSpooler.ps1` exits with code `1`, run `Diagnose-PrintError.ps1` for a full picture of what is blocking the service from starting (driver issues, corrupted spool files, RPC failures, etc.).
</Tip>

### Clearing a Stuck Print Queue

If the spooler restarts but immediately stops again, a corrupted job in the spool folder is often the cause. Clear it manually before restarting:

```powershell
# Stop the spooler, clear the queue, restart
Stop-Service -Name 'Spooler' -Force
Remove-Item -Path 'C:\Windows\System32\spool\PRINTERS\*' -Force -ErrorAction SilentlyContinue
Start-Service -Name 'Spooler'
```
