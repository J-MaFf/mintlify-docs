MAFF is an open-source toolkit for the unglamorous parts of Windows IT — the repetitive setup, the Active Directory cleanup, the reporting nobody wants to do by hand. It's five independent tools, each aimed at a specific chore: stand up a fresh workstation in a single command, remediate Active Directory in bulk, regression-test PowerShell GUI scripts, turn ClickUp tasks into clean reports, and carry a consistent Git setup across every machine you touch. Grab just the tool you need today, or run the whole suite — nothing depends on anything else.

Most tools are PowerShell-first for Windows admin work, but the ClickUp extractor and Git configuration are cross-platform and run just as happily on macOS and Linux.

## What's Included

The toolkit ships five separate tools. Pick the one that matches your immediate need, or read through all of them to understand the full scope of what's available.

## Prerequisites

Most MAFF tools share a common set of requirements. Check the individual tool documentation for anything beyond this baseline.

- **Windows 10 or Windows 11** — All PowerShell-based tools target modern Windows. The ClickUp extractor and gitconfig tools also run on macOS and Linux.
- **PowerShell 5.1 or later** — PowerShell 7\+ is preferred for all scripts. Windows PowerShell 5.1 is supported for legacy compatibility where noted.
- **Administrator rights** — The majority of scripts perform system-level operations (app installation, registry changes, Active Directory queries) and require an elevated session.
- **WinGet** — Required for `winget-app-setup`. WinGet ships with Windows 10 1809\+ and Windows 11 via the App Installer package.
- **Active Directory module (RSAT)** — Required for the AD scripts inside `PowerShellScripts`. The `ActiveDirectoryUtils` module attempts to install RSAT automatically when it detects they are missing; if elevation is unavailable it falls back to a limited read-only mode.
- **Python 3.11\+** — Required for `clickup_task_extractor` only.

<Note>
  If you import `Modules\ActiveDirectoryUtils` without elevation, the module logs a warning and continues in limited mode rather than throwing a terminating error. Elevate your session to unlock full AD write operations.
</Note>
