J-MaFf's gitconfig system gives you a consistent, opinionated Git environment on every machine you work on. It generates your `~/.gitconfig` from a version-controlled template, ships more than 40 carefully categorised aliases for everyday workflow, and keeps itself up to date automatically at each login — so you never have to think about which machine has the latest tweaks.

## What the System Does

At its core, the installer takes a single `.gitconfig.template` file that lives in this repository and renders it into a machine-specific `~/.gitconfig` with the correct local paths substituted. From that point on:

- **Aliases** give you short, memorable commands for inspecting history, committing, branching, syncing with remotes, and managing the configuration itself.
- **The interactive alias browser** (`git alias`) opens a searchable, category-tabbed TUI powered by the `textual` Python package. Press **Ctrl-G** at your shell prompt to insert a chosen alias directly onto the command line.
- **Auto-sync** runs at every login (launchd on macOS, Task Scheduler on Windows, cron on Linux). It pulls the latest commits from the gitconfig repository and, when the template has changed, regenerates `~/.gitconfig` automatically. You can also trigger it on demand with `git selfupdate`.
- **Machine-specific overrides** live in `~/.gitconfig.local`, which the installer never touches. Put per-machine settings — signing keys, local `safe.directory` entries, work-specific identity — there and they survive every update.

## Supported Platforms

| Platform | Shell / Environment | Auto-Sync Job | Minimum Requirement |
| --- | --- | --- | --- |
| macOS | bash / zsh | launchd agent | macOS 12\+, Homebrew, Python 3 |
| Linux | bash | cron job | bash 4.0\+, cron, Python 3 |
| Windows | PowerShell 5.1\+ or 7\+ | Task Scheduler | PowerShell 5.1\+, Python 3 |
